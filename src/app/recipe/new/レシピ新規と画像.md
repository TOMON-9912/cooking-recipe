# レシピ新規登録とサムネイル画像（読み方）

## 画面から見た流れ（2 ステップ）

1. **画像があるとき** … フォームが `uploadRecipeThumbnailAction` を呼ぶ → サーバーが S3 に保存 → **`path`（例: `recipes/…/….jpg`）** が返る  
2. **レシピ保存** … `createRecipeAction` にレシピ本文＋手順＋さっきの **`path`** を渡す → DB にレシピができる  

画像が無いときは 1 を飛ばして、2 だけです。

---

## サーバー側の流れ（サムネイルだけ）

```
ブラウザ（FormData に file）
    ↓
uploadRecipeThumbnailAction   … 認証・レート制限・監査ログ・バイト列の受け取り
    ↓
uploadRecipeThumbnailUsecase  … サイズ・MIME などのルールチェック
    ↓
recipeThumbnailStorageImpl    … AWS SDK で S3 に PutObject
```

**覚え方**: 玄関（action）→ ルール（usecase）→ 倉庫（infrastructure）。

---

## 主なファイル

| 役割 | ファイル |
|------|-----------|
| レシピ本体の Action | `action.ts` |
| 画像だけの Action | `upload-recipe-thumbnail-action.ts` |
| 画像のルール | `usecase/recipe/upload-recipe-thumbnail-usecase.ts` |
| 画像の定数（最大サイズ・MIME など） | `constants/recipe-thumbnail-upload.ts` |
| S3 への保存の型 | `domain/repositories/recipe/recipe-thumbnail-storage.ts` |
| S3 実装 | `infrastructure/storage/recipe-thumbnail-storage-impl.ts` |
| レート制限・監査ログ（1 ファイルにまとめたもの） | `lib/recipe-thumbnail-upload-controls.ts` |

定数を変えたいときは **`constants/recipe-thumbnail-upload.ts`** だけ見れば足ります。
