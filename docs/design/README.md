# 設計書（design）

機能追加・画面追加の**設計書**を置くディレクトリです。実装前の合意形成や、後からの振り返り用に使います。

## ファイル名

日本語で分かりやすく命名してください（例: `レシピ検索画面-設計書.md`）。

## ペアプロモードとの関係

Cursor の **ペアプロモード**（`.cursor/commands/pairpro.md`）で設計書を作る場合は、チャットに出力するだけでなく、**必ずこの `docs/design/` にマークダウンファイルとして保存**してください。

## 設計書に含める内容（初心者向けの目安）

設計書は「仕様の箇条書き」だけで終わらせず、次を含めると実装に繋がりやすくなります（詳細は `pairpro.md` のテンプレート参照）。

- **学習ゴール**（この機能で身につけたいこと）
- **実装の進め方**（どのファイルから手を付けるか、依存の向き）
- **ステップごとの完了条件**と**動作確認の仕方**
- **各ファイルの責務**（中身は文章で。コピペ用のソース全文は書かない）
- **参考ファイル**（既存の似た実装へのリンク）
- **つまずきやすい点**（ヒントのみ。答えの実装は書かない）
